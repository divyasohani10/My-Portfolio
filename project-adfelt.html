Adfelt Voice
Orchestrator

n8n Retell AI GoHighLevel Firestore Multi-Client
Overview

The Objective

Adfelt is a scalable AI-driven call automation and workflow orchestration system managing multi-client voice operations across healthcare and service-based businesses. The architecture handles appointment booking, reminders, outbound dialing, and follow-up sequences across multiple clients — all routed through a single centralized n8n instance using dynamic client ID-based logic.

The Challenge

Multi-Client Architecture Without Duplication

The fundamental challenge was supporting multiple clients with different CRM systems, webhook structures, and workflow requirements — without building a separate n8n instance for each one. Some clients like Sovereign required AxisCare CRM integration while others ran standard GHL flows. The system needed to identify which client triggered which workflow and branch accordingly, all from one central orchestration layer.

The Solution

Technical Strategy

A centralized n8n instance handles all clients using customer_id-based identification passed through webhook parameters. Conditional IF logic branches 80-90% of clients to a standard workflow (appointment booking, reminders, follow-ups) while special clients trigger additional logic. Firestore stores per-client configuration including API keys, CRM credentials, and metadata — keeping the core workflow clean. GHL triggers webhooks to n8n, n8n orchestrates the Retell AI voice interaction, call outcomes are sent back to n8n, and CRM updates plus follow-up scheduling are executed automatically.

Key Results

Impact

  • Multiple clients managed from single n8n instance
  • 80–90% standardization with conditional exception handling
  • Dynamic client config stored and fetched from Firestore
  • Full call lifecycle automated: trigger → voice → outcome → CRM update
